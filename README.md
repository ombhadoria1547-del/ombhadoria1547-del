### 📖 Overview

A production-style distributed rate limiter built using **Go**, **Redis**, and **Lua** that efficiently controls request throughput in distributed environments.

The system supports multiple rate limiting algorithms while ensuring atomic state updates through Redis Lua scripting, making it suitable for horizontally scalable backend services.

---

### ✨ Key Features

- 🚀 Redis-backed **Token Bucket**
- 🚀 Redis **Sliding Window** algorithm
- ⚡ Atomic operations using **Lua Scripts**
- 🔐 Dynamic client configuration through Admin APIs
- 📦 Dockerized deployment
- 📊 Standard HTTP Rate-Limit headers
- ⚙️ Configurable rate & burst limits
- 📈 Load tested using **k6**
- 🌐 Deployed using **Render**
